TERMINAL COMMANDS:

Certain functions are accessable via "terminal style" commands.
Enter the keyword directly into the "To Device" text box.

help

Open this help window. Example "help"

clear

clear the terminal 

scan

scan for avaliable ports. Ex: "scan"

con

Connect to the selected ports. Ex: "con"

dcon 

Disconnect from the active port. Ex: "dcon"

auto

toggle the auto-reconnect. Ex: "auto"

com [port#]

Connect to com"port#". Ex: com 50 (connects to com50)
Optional args:

-baud [baudrate] use baudrate [baudrate]. Ex: com 50 -baud 9600

-xonxoff enable xonxoff control. Ex: com 50 -xonxoff

-rtscts enable rtscts. Ex: com 50 -rtscts 

-dsrdtr enable dsrdtr. Ex: com 50 -dsrdtr

-auto enable auto-reconnect 

log

Open the latest log file. Ex: log

Optional args:

-o open a log file from the log directory. Ex: log -o

-a archive the latest file and start a new one. Ex: log -a

-n [name] name the current port "name". Ex: log -n myName

-h display help for logs 

script

run a script. Ex: script 

optional arguments: 

-o [name] open a script from the script directory. If a name is provided, it will be the one selected. Ex: script -o myscript

-r run the script after opening 

-s [name] save the current script. If a name is provided, it will be saved as "name".txt. Ex: script -s myscript 

-t switch over to the script tab. Ex: script -t

-n [name] create a new script. If a name is provided, it will be automatically saved as that name. Ex: script -n myscript

-h prints help for the script command. Ex: script -h

 

 

Command Conflicts:

If there are possible conflicts between commands and messages to send to a device, a command char can be used to distinguish commands.
For example, "$" would mean that only commands that begin with $ are accepted ($help, $new, etc...).
Set a command char with:

Adv. Settings > Commands

Scripts

A script is an easy way to automate an interface with a device.

Each new line of the script is run as an input in the "to device" text box. If lines are NOT commands, they are sent to the serial device after a delay time. This is to give the serial device time to respond or do something. 

Type line will be evaluated and sent one after the other. 


Forced commands: 
Lines that start with a "#" are forced to be interpreted as commands.
Scripts extend the standard Serial Killer commands with script-specific functions:

#name=[name]: saves the script as "name".txt every time it is run. 
#delay=[number] : set the delay between lines to [number]. Ex: #delay=200: wait 200ms between lines
#loop=[number] : loops the section between itself and #endloop. If a number is provided, it will loop [number] of times.
If no number is provided "ex. #loop". It will loop infinitely. 
#endloop: ends the loop above
#stop: stops the script at this line

Example script: 

#name=myscript
#delay=500
i am a script!!!
#loop=10
hello
world
#endloop
#clear

This script saves itself as "myscript.txt" to the script directory. 
It then sends "i am a script!!!" and delays 500 milliseconds
It then sends "hello", then delays 500 milliseconds
It then sends "world", then delays 500 milliseconds. 
It repeats sending "hello" and "world" 4 additional times. 
It then clears the terminal.
